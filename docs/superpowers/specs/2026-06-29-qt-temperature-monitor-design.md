# LoRa 温度实时监测 Qt 应用 — 设计文档

日期：2026-06-29
状态：待评审

## 1. 概述

开发一个跨平台（Linux / Windows）的 Qt6 C++ 桌面应用，通过串口 Modbus RTU 协议实时读取 LoRa 温度集中器连接的温度传感器数据，实时显示并自动保存到 CSV 文件。

应用面向项目目录中的"集中器模块 ModbusRtu 协议"（`集中器ModbusRtu协议260330_EID.docx`）。

## 2. 需求

### 2.1 功能性需求

- 通过串口 Modbus RTU 读取集中器的温度1寄存器（起始地址 `0x76C1`）
- 实时显示当前温度：当前值卡片 + 实时折线图 + 数据表格
- 自动将采集数据追加写入 CSV 文件（每次开始采集生成新文件）
- 上下限报警：超出阈值时卡片变色 + 状态栏提示
- 串口参数、节点配置、阈值可配置并持久化

### 2.2 非功能性需求

- 跨平台：Linux + Windows
- 采样周期默认 2 秒（可配置）
- 默认监测 ID1、ID2 两个节点（节点数量可调，1~16）
- UI 在采集期间不卡顿（采集在子线程）

### 2.3 当前实施范围（明确参数）

| 参数 | 值 |
|------|-----|
| 读取寄存器 | 温度1，起始 `0x76C1` |
| 功能码 | 0x04（读输入寄存器） |
| 节点数量 | 2（ID1、ID2），UI 可调 1~16 |
| 采样周期 | 2000 ms |
| 数据解析 | Modbus 大端传输；`quint16` → `int16`（补码）→ ÷10.0 = ℃ |
| 串口格式 | 8 数据位 / 1 停止位 / 无校验 / 默认 9600 bps |

## 3. 协议细节（来自项目协议文档）

- **通讯格式**：8 数据位、1 停止位、无奇偶校验、波特率可设（默认 9600bps）
- **温度1寄存器**：起始地址 `0x76C1`（30401），1~200 号节点连续映射，每节点 1 个寄存器
- **单位**：0.1℃，负数采用补码（如 -16.0℃ 传输码为 `0xFF60`）
- **字节序**：Modbus RTU 天然大端（高字节先传）

### 3.1 读取示例

读取 ID1、ID2 的温度1（2 个寄存器）：
- 主机发送：`01 04 76 C1 00 02 + CRC`
- 从机回复：`01 04 04 <T1_H> <T1_L> <T2_H> <T2_L> + CRC`

### 3.2 温度解析

```
raw = quint16 值（QModbusDataUnit 已解析字节序）
temp_int16 = qint16(raw)        // 解释补码，处理负温度
temp_celsius = temp_int16 / 10.0
```

## 4. 整体架构

```
┌─────────────────────────────────────────────────┐
│                  MainWindow                      │
│  ┌──────────┐  ┌──────────────────────────────┐ │
│  │ 配置面板  │  │  当前值卡片网格 (按节点)      │ │
│  │ 串口参数  │  ├──────────────────────────────┤ │
│  │ 节点设置  │  │  实时折线图 (QtCharts)        │ │
│  │ 报警阈值  │  ├──────────────────────────────┤ │
│  │ 开始/停止 │  │  数据表格 (滚动)             │ │
│  └──────────┘  └──────────────────────────────┘ │
│                  状态栏                          │
└─────────────────────────────────────────────────┘
         ▲ dataReady(QVector<Sample>)
         │ 信号槽（跨线程 Queued 自动连接）
┌────────┴────────┐  ┌──────────────┐  ┌────────────┐
│  ModbusWorker   │  │  CsvWriter   │  │ ChartManager│
│  (QThread 子线程)│  │  (主线程写)   │  │ (主线程更新) │
│  每 2s 轮询 0x04│  │              │  │            │
└─────────────────┘  └──────────────┘  └────────────┘
```

**核心解耦**：`ModbusWorker` 在子线程定时轮询，每次采集完发 `dataReady` 信号；主线程的 UI / 图表 / CSV 各自独立响应同一信号，互不阻塞。

## 5. 模块划分

| 模块 | 职责 | 关键接口 |
|------|------|----------|
| `ModbusWorker` | 子线程轮询 Modbus，解析温度1 | `start()/stop()`；信号 `dataReady(QVector<Sample>)`、`error(QString)` |
| `Sample` | 单次采集数据结构 | `timestamp, nodeId, raw, tempCelsius, online` |
| `CsvWriter` | 追加写 CSV，按会话切分文件 | `startSession(dir)/write(QVector<Sample>)/close()` |
| `ChartManager` | 管理 QtCharts 曲线（每节点一条） | `append(QVector<Sample>)/setNodeVisible(id)` |
| `MainWindow` | 组装 UI、连接信号槽、配置管理 | 串口/节点/报警配置 UI |
| `AppConfig` | 配置持久化（QSettings） | 串口参数、节点数、阈值、CSV 目录 |

### 5.1 模块边界说明

- `ModbusWorker` 只负责"读寄存器 → 解析成 `Sample` → 发信号"，不关心 UI/文件
- `CsvWriter` 只负责"收 Sample → 写文件"，不关心数据来源
- `ChartManager` 只负责"收 Sample → 更新曲线"，不关心存储
- `MainWindow` 负责把三者用信号槽连起来，并管理配置
- 每个模块可独立理解和测试

## 6. 数据流

1. 用户点"开始" → `MainWindow` 读取配置 → `CsvWriter` 创建新文件（`temp_YYYYMMDD_HHMMSS.csv`）→ 启动 `ModbusWorker`
2. `ModbusWorker` 每 2000ms：
   - 用 `QModbusRtuSerialMaster` 发送 0x04 请求，起始 `0x76C1`，读 N 个寄存器
   - 解析每个寄存器：`qint16(raw) / 10.0` → 温度℃
   - 组装 `QVector<Sample>` 发 `dataReady` 信号
3. 主线程收到 `dataReady`：
   - **CsvWriter**：每个 Sample 追加一行
   - **ChartManager**：每节点曲线 append 新点（保留最近 30 分钟，约 900 点）
   - **卡片/表格**：更新当前值，超阈值变色
4. 用户点"停止" → `ModbusWorker` 停止轮询 → `CsvWriter` 关闭文件

## 8. CSV 格式

- 编码：UTF-8 with BOM（保证 Excel 正确识别）
- 文件名：`temp_YYYYMMDD_HHMMSS.csv`，位于用户配置的 CSV 目录
- 表头：

```
timestamp,node_id,temp_celsius,raw,online,alarm
2026-06-29 14:30:01.123,1,19.1,191,1,0
2026-06-29 14:30:01.123,2,25.3,253,1,0
```

- `alarm`：0 正常，1 超上限，-1 超下限
- `raw`：寄存器原始十进制值（便于核对）

## 10. 错误处理

- 串口打开失败 → 状态栏红字提示，不启动采集
- Modbus 请求超时 / 异常响应 → 状态栏提示，本次采集跳过（不写无效数据），下次周期重试
- 连续 3 次失败 → 状态栏红字提示"通讯异常"，继续重试
- `Sample.online` 字段含义：本次 Modbus 请求成功时，所有请求到的节点 `online=1`；请求失败时整批丢弃，不写入 CSV（不做单节点离线判断，因为不读节点类型寄存器无法可靠区分"0℃"与"节点不存在"）
- CSV 写入失败 → 弹窗提示，停止采集

## 11. 不做的事（YAGNI）

- 不做历史回放 / 数据库 / Excel 导出（后续按需加）
- 不做 4G/LoRaWan 云平台对接
- 不做集中器配置（用项目里现有的 `LoraConcentratorConfig.exe`）
- 不读温度2/温度3/湿度/电流（当前只有温度1需求）
- 不读节点类型寄存器（简化：直接读温度，离线判断靠返回值）

## 12. 技术栈与依赖

- **语言**：C++17
- **框架**：Qt6（Core / Gui / Widgets / SerialPort / SerialBus / Charts）
- **构建**：CMake
- **Modbus**：`QtSerialBus` 的 `QModbusRtuSerialMaster`（官方模块，跨平台，无需外部库）
- **图表**：`QtCharts`

### 12.1 依赖安装说明

- Linux：`qt6-base-dev qt6-serialport-dev qt6-serialbus-dev qt6-charts-dev cmake`
- Windows：Qt6 在线安装器勾选 Serial Port / Serial Bus / Charts 模块
